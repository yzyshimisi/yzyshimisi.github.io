排序与二分查找

快速排序

算法基本步骤

1.从序列中挑一个元素x，作为分界点。
2.调整序列为两段，使得左段中所有元素<=x，右段中所有元素>=x。
3.分别递归处理左右两端

时间复杂度O(nlogn)

归并排序

算法基本步骤

1.将含有n个元素的序列平分成两个子序列

2.递归调用归并排序，使两个子序列有序
3.合并这两个有序的子序列

时间复杂度O(nlogn)

模板代码

桶排序（Bucket sort）

又称箱排序，将待排序序列基于某种映射函数划分到不同的桶中，在桶内进行比较排序后枚举输出。

算法基本步骤

1.根据待排序序列中最大元素和最小元素的差值范围和映射规则，确定申请桶的个数。

2.遍历待排序序列，将每个元素移动到对应的桶中。
3.对每一个桶中的元素进行排序。
4.不是空的桶里把排好序的元素顺序拼接起来。
时间复杂度：O(n+k)

计数排序（Counting Sort）

将输入的数据值转化为键存储在额外开辟的数组空间中，利用数组下标来确定元素的正确位置。计数排序要求输入的数据必须是有确定范围的整数。

算法基本步骤

1.找出待排序序列中的最大值max、最小值min。
2.创建一个新的数组C，其长度range是max-min+1，其默认元素值都为0。
3.遍历待排序序列的元素，元素值 i 的出现次数，存入数组C的第[i-min]项
4.重新填充待排序序列：

遍历数组C，根据数组C[i]的值（即根据待排列元素出现的次数）重复地将数组C下标i+min，按顺序赋值到原序列，即能完成排列。

处理过程（例子：[27, 28, 68, 45, 90, 5, 79]）

1）遍历待排序序列，找到最大值90，最小值5
2）创建新数组C，其长度是86(90-5+1)，各元素默认值为0
3）遍历待排序序列的各元素，27出现1次，数组C第[27-5]项的值加1，即元素27
出现的次数加1；28出现1次，数组C第[28-5]项的值加1；以此类推，遍历结束
后数组C中的各元素值为[…]
4）遍历数组C，数组C[0]的值为1，则将0+5赋值作为原序列的首个元素，以此类推
遍历结束后，得到序列：5，27，28，45，68，79，90。

时间复杂度

O(n+k)

4.Hexo主题切换

以NexT主题为例

不同的主题具体可以访问官方文档来进行学习

1.安装next主题

推荐使用npm的方式安装这个主题

不推荐使用git来安装因为在一个用 git 管理的项目文件夹下是不应该出现另一个用 git 管理的项目的，否则会导致 git 提交时出现问题

也可以在GitHub上搜索next主题的团队，下载next主题ZIP包，不要直接git clone

2.将得到的文件夹重命名为next并复制到博客根目录下的 themes 文件夹中

3.修改站点配置文件(_config.yml)

4.启用NexT主题

在站点配置文件找到theme属性，把值改成next即可（注意冒号与next之间有空格）

5.修改主题配置文件（_config.yml）

设定主题风格Scheme 是 NexT 提供的一种特性，借助于 Scheme，NexT 为你提供多种不同的外观

Scheme 的切换通过更改主题配置文件，搜索 scheme 关键字。 你会看到有三行 scheme 的配置，将你需用启用的 scheme 前面注释 # 去除即可。

设定博客主标题上方的logo

custom_logo: /

设置头像

修改字段 avatar，值设置成头像的链接地址。其中，头像的链接地址可以是：
完整的互联网URL http://baidu/avatar.png

站点内的地址        将头像放置主题目录下的 source/uploads/ （若不存在则新建 uploads 目录）
配置为：avatar: /uploads/avatar.png
或者 放置在 source/images/ 目录下
配置为：avatar: /images/avatar.png

设置侧栏

可以通过修改 sidebar 字段来控制侧栏的行为（位置、显示时机）
修改 sidebar.position 的值
left - 靠左放置
right - 靠右放置
修改 sidebar.display 的值
post - 默认行为，在文章页面（拥有目录列表）时显示
always - 在所有页面中都显示
hide - 在所有页面中都隐藏（可以手动展开）
remove - 完全移除
